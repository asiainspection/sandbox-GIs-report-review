## How to read this document

Each rule entry contains:
- **ID** — unique rule identifier (section.subsection.rule)
- **Field / Location** — where to look in the report
- **What to check** — the exact verification to perform
- **Scope** — `QUESTION` (check only that field), `SECTION` (cross-check within the same checklist section), `FULL REPORT` (cross-check across multiple sections or documents)
- **Error example** — a concrete example of what a wrong value looks like
- **Correct example** — a concrete example of the correct value looks like
- **Severity** — `BLOCKING` or `MINOR`, or `TO CONFIRM` for ambiguous rules

## Section 2 – Quantities and Production Status

---
**ID:** 2.1.1
**Field / Location:** PO & Product details — Produced/Packed quantities (PSI)
**What to check:** Apply the production-status rule table:
- 100% finished, 50–80% packed → perform inspection.
- <100% finished, 50–80% packed → perform inspection.
- 100% finished, <50% packed → perform inspection.
- <100% finished AND <50% packed → inform the quality team on-site immediately and issue a missed inspection/abort.
**Scope:** `SECTION` — cross-check finished %/packed % against this table.
**Error example:** Inspection proceeded normally at 20% finished / 15% packed, with no missed-inspection notice.
**Correct example:** Inspection aborted and quality team informed when both thresholds fall below the minimums above.
**Severity:** `BLOCKING`


```check
where: [product._first.real_packed_quantity]
when: null
check: null
```

---
**ID:** 2.1.2
**Field / Location:** Inspector's remark / Summary review — partial packed/produced quantity mismatch (e.g. booked 80% PSI, actual 20%)
**What to check:** Proceed with the inspection, add an important remark, fail the quantity checkpoint, and mention this on the draft report signed by the factory. Sample size must be selected only from finished and packed goods. If there is no finished/packed product, workmanship check is not required, but the inspector must still check what is checkable.
**Scope:** `FULL REPORT` — cross-check remark, quantity checkpoint result, and draft report content.
**Error example:** Quantity checkpoint marked Pass despite a large gap between booked and actual finished/packed percentage, with no remark.
**Correct example:** Remark present, quantity checkpoint marked Fail, sample drawn only from finished/packed stock.
**Severity:** `BLOCKING`

**ID:** 5.1.3
**Field / Location:** Measurement file — Product measurement sequence and method
**What to check:** Product measurements recorded in the order Height (H) x Width (W) x Depth (D):
- Height: lowest to highest point, vertically.
- Width: side to side, horizontally.
- Depth: front to back, horizontally.
- Always measure the longest dimension for each axis; width is generally longer than depth.
- If placed against a wall, depth is the distance from the wall to the front of the product.
**Scope:** `SECTION` — cross-check recorded H/W/D values and sequence against product type and the reference diagram.
**Error example:** Product recorded as W x D x H instead of H x W x D.
**Correct example:** Product recorded strictly as H x W x D, consistent with the reference diagram for that furniture type.

**ID:** 7.2.3
**Field / Location:** Tests & checkpoints — Drop height by weight class
**What to check:** ISTA 3A: <32kg vs 32–70kg product+box weight determines drop height (e.g. most drops = 460mm/300mm; hazard drops 8 and 16 = 910mm/600mm; Step 5a/5b hazard block drop = 400mm regardless of weight). ISTA 2C: drop height depends on weight class (0–23kg/23–45kg/>45kg) and level (Level 2 vs Level 3), e.g. Level 3 0–23kg = 460mm, Level 2 0–23kg = 610mm.
**Scope:** `SECTION` — cross-check declared product+box weight, drop height used, and ISTA level/standard.
